1//! # Module `context::calculators`
2//!
3//! Built-in [`ContextCalculator`] implementations for common PDE quantities.
4//!
5//! These calculators cover the most frequent context variables in
6//! transport‚Äìreaction‚Äìdiffusion problems. Each implements [`ContextCalculator`]
7//! and can be registered directly in [`SolverConfiguration`].
8//!
9//! ## Available calculators
10//!
11//! | Type | Provides | Notes |
12//! |---|---|---|
13//! | [`TimeCalculator`] | `ContextVariable::Time` | Current simulation time |
14//! | [`TimeStepCalculator`] | `ContextVariable::TimeStep` | Current `dt` |
15//! | [`FDGradientCalculator`] | `ContextVariable::SpatialGradient` | FD gradient (fwd/ctr/bwd) |
16//! | [`FDLaplacianCalculator`] | `ContextVariable::External { name }` | FD Laplacian |
17//! | [`TrapezoidalIntegral`] | `ContextVariable::External { name }` | Spatial quadrature |
18//! | [`ExternalTabulated`] | `ContextVariable::External { name }` | Tabulated f(t) interpolation |
19//!
20//! ## Mesh ownership
21//!
22//! Spatial calculators ([`FDGradientCalculator`], [`FDLaplacianCalculator`],
23//! [`TrapezoidalIntegral`]) hold an [`Arc<dyn Mesh>`] internally. This is an
24//! implementation detail consistent with INV-1 (DD-007): the mesh never appears
25//! in the public `ContextCalculator` API. At v0.5.0, `Arc<dyn Mesh>` will be
26//! replaced by a concrete [`DiscreteOperator`] (INV-2, DD-012), with zero change
27//! to the `ContextCalculator` trait signature.
28//!
29//! [`ContextCalculator`]: crate::context::calculator::ContextCalculator
30//! [`SolverConfiguration`]: crate::solver::config::SolverConfiguration
31//! [`Arc<dyn Mesh>`]: std::sync::Arc
32//! [`DiscreteOperator`]: crate::operators

48/// Finite-difference stencil for first-order spatial derivatives.
49///
50/// Used by [`FDGradientCalculator`]. At interior nodes all three schemes are
51/// fully applicable; at boundary nodes the calculator falls back to the nearest
52/// available one-sided stencil automatically.
53///
54/// | Variant | Stencil | Order | Boundary fallback |
55/// |---|---|---|---|
56/// | `Forward` | `(u[i+1] ‚àí u[i]) / dx` | 1st | `Backward` at last node |
57/// | `Backward` | `(u[i] ‚àí u[i‚àí1]) / dx` | 1st | `Forward` at first node |
58/// | `Central` | `(u[i+1] ‚àí u[i‚àí1]) / (2 dx)` | 2nd | One-sided at boundaries |
59///
60/// # Examples
61///
62/// ```rust
63/// use oxiflow::context::calculators::FDScheme;
64///
65/// let scheme = FDScheme::Central;
66/// ```
67#[non_exhaustive]
68#[derive(Debug, Clone, PartialEq, Eq)]
69pub enum FDScheme {
70    /// Forward difference: `(u[i+1] ‚àí u[i]) / dx` ‚Äî 1st-order accurate.
71    Forward,
72    /// Backward difference: `(u[i] ‚àí u[i‚àí1]) / dx` ‚Äî 1st-order accurate.
73    Backward,
74    /// Central difference: `(u[i+1] ‚àí u[i‚àí1]) / (2 dx)` ‚Äî 2nd-order accurate.
75    Central,
76}
