Expand description

Declares context variable requirements for any engine component.

Implemented by PhysicalModel, BoundaryCondition (J2), SourceTerm, and CouplingOperator (J3). The solver aggregates declarations from all components before solving and guarantees every required variable is available.

required_variables() has no default — every implementor must declare its requirements explicitly, even if that declaration is vec![].

fn required_variables(&self) -> Vec<ContextVariable>

Variables this component must have. Solver raises OxiflowError::MissingCalculator if any is absent.

No default — explicit declaration is mandatory.

fn depends_on(&self) -> Vec<ContextVariable>

Variables that must be computed before this component runs.

Used by chain.rs for topological ordering (DD-009, J2).

Source

fn priority(&self) -> u32

Execution priority within the calculator chain (lower = earlier).

Reserved ranges: 0 = system (Time, TimeStep), 50 = external data, 100 = default for derived quantities.
